* Used for **non-blocking operations**

## 1ï¸âƒ£ Callbacks

**Definition**

* A function passed as an argument
* Executed **after** an async task completes

**Syntax**

```js
function task(callback) {
  callback();
}
```

**Pros**

* Simple concept
* Works in older JS

**Cons**

* Callback Hell (nested code)
* Hard error handling
* Poor readability

**Keyword**

* `callback()`

---

## 2ï¸âƒ£ Promises

**Definition**

* Object representing a value available **now, later, or never**

**States**

* Pending
* Fulfilled (`resolve`)
* Rejected (`reject`)

**Syntax**

```js
promise.then().catch();
```

**Pros**

* Better readability than callbacks
* Chainable
* Centralized error handling

**Cons**

* Still slightly verbose

**Keywords**

* `new Promise`
* `.then()`
* `.catch()`
* `.finally()`

---

## 3ï¸âƒ£ Async / Await

**Definition**

* Modern way to handle promises
* Makes async code look synchronous

**Rules**

* `await` works only inside `async` functions
* Built on top of promises

**Syntax**

```js
async function func() {
  const data = await promise;
}
```

**Pros**

* Clean and readable
* Easy debugging
* Best practice today

**Cons**

* Needs modern JS support

**Keywords**

* `async`
* `await`
* `try...catch`
