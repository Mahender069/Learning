## 1. What is EventEmitter?

* **EventEmitter** is a core Node.js class
* Used to create and handle **custom events**
* Part of the built-in **`events`** module
* Works on **event-driven architecture**

---

## 2. Why EventEmitter?

* Node.js is **non-blocking** and **asynchronous**
* EventEmitter helps different parts of an app **communicate**
* Makes code **modular** and **clean**

---

## 3. Basic Terms

| Term     | Meaning                               |
| -------- | ------------------------------------- |
| Event    | Something that happens                |
| Listener | Function that runs when event happens |
| Emit     | Trigger an event                      |

---

## 4. Importing EventEmitter

```js
const EventEmitter = require('events');
```

---

## 5. Creating EventEmitter Object

```js
const emitter = new EventEmitter();
```

---

## 6. Listening to Events

### `on()`

```js
emitter.on('event', () => {});
```

### `once()`

* Runs **only one time**

```js
emitter.once('event', () => {});
```

---

## 7. Emitting Events

```js
emitter.emit('event');
```

* Can pass data as arguments

```js
emitter.emit('event', data);
```

---

## 8. Multiple Listeners

* One event can have **many listeners**
* They execute in the **order added**

```js
emitter.on('event', () => console.log('1'));
emitter.on('event', () => console.log('2'));
```

---

## 9. Removing Listeners

### `removeListener()`

```js
emitter.removeListener('event', listenerFn);
```

### `removeAllListeners()`

```js
emitter.removeAllListeners('event');
```

---

## 10. Using EventEmitter with Classes

### Extending EventEmitter

```js
class MyClass extends EventEmitter {
  constructor() {
    super();
  }

  start() {
    this.emit('start');
  }
}
```

---

## 11. Wrapping `emit()` in Methods (Best Practice)

* Do **not** call `emit()` directly from outside
* Wrap it inside a meaningful method

```js
process() {
  // logic
  this.emit('done');
}
```

---

## 12. Passing Data with Events

```js
emitter.on('login', (username) => {
  console.log(username);
});

emitter.emit('login', 'Rahul');
```

---

## 13. Synchronous Nature

* EventEmitter runs **synchronously**
* Listeners block execution if they are slow

---

## 19. Best Practices

* Use clear event names
* Keep listeners small
* Use events for **notifications**, not logic
* Prefer methods over direct `emit()`
