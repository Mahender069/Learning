## 1ï¸âƒ£ What is a Route?

> **A route defines how an application responds to a client request for a specific URL and HTTP method.**

Example:

```js

  res.send("Users list");
});
```

## 2ï¸âƒ£ Parts of a Route

* `METHOD` â†’ HTTP method (`GET`, `POST`, `PUT`, `DELETE`)

## 8ï¸âƒ£ Express Router (Modular Routes)

Used to organize routes.

### `routes/userRoutes.js`

```js

const router = express.Router();

router.get("/", (req, res) => {
  res.send("All users");
});

module.exports = router;
```

### `app.js`

```js

app.use("/users", userRoutes);
```

## 9ï¸âƒ£ Route Order Matters âš ï¸

Express matches routes **top to bottom**.

```js
app.get("/users/:id", ...)
app.get("/users/profile", ...) // âŒ unreachable
```

Correct order:

```js
app.get("/users/profile", ...)
app.get("/users/:id", ...)
```

ðŸ‘‰ What is dynamic routing?

Dynamic routing means creating routes that accept changing (dynamic) values in the URL.

These values are called route parameters.

Basic example
app.get("/users/:id", (req, res) => {
  res.send("User ID: " + req.params.id);
});


Requests:

/users/1
/users/25
/users/abc


All match the same route.

How it works

:id is a dynamic parameter

Express captures its value

Value is available in req.params

req.params.id

Multiple dynamic parameters
app.get("/products/:category/:pid", (req, res) => {
  const { category, pid } = req.params;
  res.send(category + " " + pid);
});


Request:

/products/mobile/101
