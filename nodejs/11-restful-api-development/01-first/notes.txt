A **RESTful API** is an API (Application Programming Interface) that follows the principles of **REST**, which stands for **Representational State Transfer**. It‚Äôs a common way for software systems to communicate over the internet, especially web and mobile apps.

---

## What REST Means (in simple terms)

REST is a **design style** for building APIs that use standard web technologies‚Äîmainly **HTTP**.

A RESTful API lets clients (like browsers or mobile apps) **request, create, update, or delete data** from a server using predictable URLs and HTTP methods.

---

## Key Concepts of a RESTful API

### 1. Resources

Everything is treated as a **resource** (users, products, orders, etc.).

Example:

```
/users
/products/123
```

---

### 2. HTTP Methods (CRUD operations)

| HTTP Method | Action      | Example           |
| ----------- | ----------- | ----------------- |
| GET         | Read data   | Get user details  |
| POST        | Create data | Create a new user |
| PUT / PATCH | Update data | Update user info  |
| DELETE      | Delete data | Remove a user     |

Example:

```
GET /users/1
DELETE /users/1
```

---

### 3. Stateless

Each request is **independent**.

* The server does **not remember** previous requests
* All required information (like authentication tokens) must be sent with every request

---

### 4. Data Format (usually JSON)

RESTful APIs typically exchange data in **JSON** format.

Example response:

```json
{
  "id": 1,
  "name": "Rahul",
  "email": "rahul@example.com"
}
```

---

### 5. Uses HTTP Status Codes

| Code | Meaning      |
| ---- | ------------ |
| 200  | Success      |
| 201  | Created      |
| 400  | Bad request  |
| 401  | Unauthorized |
| 404  | Not found    |
| 500  | Server error |
