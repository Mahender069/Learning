## 3ï¸âƒ£ What is GraphQL?

### Definition

* GraphQL is a **query language for APIs**
* Lets the **client ask for exactly the data it wants**

### Key benefits

* No overfetching

* One request for complex data
* Client controls response shape

## 1ï¸âƒ£4ï¸âƒ£ What is a Resolver?

## 1ï¸âƒ£7ï¸âƒ£ What is `_` (underscore)?

* `_` means **parameter exists but is unused**
* Common JS convention

```js

```

Means:

```js
(parent, args)
```

but parent is ignored.

---

## 1ï¸âƒ£8ï¸âƒ£ What is `parent`?

### Definition

* `parent` is the **result returned by the resolver above**

### Used in nested fields

```graphql

    title
  }
}
```

Resolvers:

```js

  posts: (parent) => getPosts(parent.id)

```

## 1ï¸âƒ£9ï¸âƒ£ Resolver Flow (Very Important)

2. Top-level resolver runs
3. Result becomes `parent`
4. Nested resolvers run
5. Response is shaped
