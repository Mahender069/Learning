# 📘 JavaScript Event Loop + Node.js Notes

---

# 1️⃣ JavaScript is Single Threaded

JavaScript can run **only one task at a time**.

Execution order:

```
Top → Bottom
One line at a time
```

Example:

```javascript
console.log("A");
console.log("B");
console.log("C");
```

Output:

```
A
B
C
```

---

# 2️⃣ Asynchronous Tasks

Some tasks take time:

Examples:

```
setTimeout
API requests
file reading
user events
```

Example:

```javascript
setTimeout(() => {
  console.log("B");
}, 2000);

console.log("A");
```

Output:

```
A
B
```

JS does **not wait** for async tasks.

---

# 3️⃣ JavaScript Runtime Components

Main parts:

```
Call Stack
Web APIs
Callback Queue
Event Loop
```

---

## Call Stack

Where JS executes functions.

```
push → run → pop
```

Example:

```
console.log()
function calls
```

---

## Web APIs

Handled by **browser or Node runtime**.

Examples:

```
setTimeout
fetch
DOM events
```

Async tasks run here.

---

## Callback Queue

When async tasks finish, their **callbacks go here**.

```
callback → queue → wait
```

---

## Event Loop

Continuously checks:

```
Is call stack empty?
```

If yes:

```
Move callback from queue → stack
```

---

# 4️⃣ Event Loop Flow

```
Code
 ↓
Call Stack
 ↓
Web APIs
 ↓
Queue
 ↓
Event Loop
 ↓
Call Stack
```

---

# 5️⃣ Microtasks vs Macrotasks

JavaScript has **two task queues**.

| Queue           | Examples                     |
| --------------- | ---------------------------- |
| Microtask Queue | Promise.then, queueMicrotask |
| Macrotask Queue | setTimeout, setInterval      |

---

## Priority Order

```
1. Synchronous Code
2. Microtasks
3. Macrotasks
```

---

## Example

```javascript
console.log("Start");

setTimeout(() => console.log("Timeout"), 0);

Promise.resolve().then(() => console.log("Promise"));

console.log("End");
```

Output:

```
Start
End
Promise
Timeout
```

---

# 6️⃣ Event Loop Cycle

Event loop runs like this:

```
Run sync code
Run all microtasks
Run one macrotask
Repeat
```

---

# 📘 Node.js Event Loop Phases

Node has **extra phases**.

Order:

```
Timers
Pending Callbacks
Idle / Prepare
Poll
Check
Close Callbacks
```

Loop repeats continuously.

---

## Timers Phase

Runs:

```
setTimeout
setInterval
```

---

## Pending Callbacks

Handles **system callbacks** like network errors.

---

## Idle / Prepare

Used internally by Node.

---

## Poll Phase (Important)

Handles **I/O operations**:

```
file reading
database
network
```

Example:

```javascript
fs.readFile("file.txt", () => {});
```

---

## Check Phase

Runs:

```
setImmediate()
```

---

## Close Callbacks

Runs cleanup events.

Example:

```
socket.close()
```

# 📦 Buffers in Node.js

## What is a Buffer?

A **Buffer is temporary memory used to store binary data**.

```
Buffer = binary data storage
```

Used when working with:

```
files
images
videos
network data
streams
```

---

## Example

```javascript
const buffer = Buffer.from("Hello");
console.log(buffer);
```

Output:

```
<Buffer 48 65 6c 6c 6f>
```

Convert back:

```javascript
buffer.toString()
```

---

## Buffer Creation

```
Buffer.from()
Buffer.alloc()
Buffer.allocUnsafe()
```

---

## Real Uses

Buffers appear in:

```
file reading
image uploads
network responses
streams
encoding
```

# 🌊 Streams in Node.js

## What is a Stream?

A **stream processes data in small chunks instead of loading everything at once**.

```
Stream = chunk by chunk data processing
```

---

## Why Streams

Without stream:

```
Load full file → memory heavy
```

With stream:

```
file → chunk → chunk → process
```

Better for:

```
large files
videos
downloads
uploads
```

---

## Example

```javascript
const fs = require("fs");

const stream = fs.createReadStream("file.txt");

stream.on("data", chunk => {
  console.log(chunk);
});
```

Each **chunk is a Buffer**.

---

# Types of Streams

| Type      | Purpose      |
| --------- | ------------ |
| Readable  | Read data    |
| Writable  | Write data   |
| Duplex    | Read + Write |
| Transform | Modify data  |

---

## Readable Stream

```
fs.createReadStream()
```

---

## Writable Stream

```
fs.createWriteStream()
```

---

## Duplex Stream

Example:

```
TCP sockets
```

---

## Transform Stream

Used for:

```
compression
encryption
```

Example module:

```
zlib
```

---

# 🔗 Stream Piping

Streams can connect together.

Example:

```javascript
fs.createReadStream("input.txt")
  .pipe(fs.createWriteStream("output.txt"));
```

Flow:

```
input → stream → output
```
